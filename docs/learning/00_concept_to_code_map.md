# 00 Concept to Code Map

This map links every business/pharma concept to its AWS target architecture,
local MVP equivalent, and the specific Python code that implements it.

Use this as your navigation guide when reading code or explaining Areos to customers.

---

## Core concept map

| Business concept | Pharma/OT meaning | AWS target | Local MVP | Code |
|---|---|---|---|---|
| **Tenant** | Customer / company (e.g., Acme Pharma) | AWS account or logical tenant isolation | `tenant_id` string on every model | `models/asset.py`, `standards/isa95.py` |
| **Enterprise** | Corporate entity owning one or more sites | SiteWise asset hierarchy root | `Enterprise` model | `standards/isa95.py` |
| **Site** | Manufacturing site (e.g., Hyderabad) | SiteWise asset hierarchy level 1 | `Site` model | `standards/isa95.py`, `simulation/plant_topology.py` |
| **Area** | Manufacturing area (e.g., OSD Manufacturing) | SiteWise asset hierarchy level 2 | `Area` model | `standards/isa95.py` |
| **Room** | GMP manufacturing room (Grade D, ISO 8, etc.) | SiteWise asset | `Room` model | `standards/isa95.py` |
| **AHU** | HVAC utility asset serving a room | SiteWise asset | `Equipment(type=ahu)` | `standards/isa95.py`, `models/utility.py` |
| **Tablet press** | Production equipment | SiteWise asset | `Equipment(type=tablet_press)` | `standards/isa95.py` |
| **Batch** | Manufactured lot — product + procedure + operation + phase | SiteWise context / DynamoDB | `BatchRecord` | `standards/isa88.py` |
| **Humidity tag** | BMS/EMS relative-humidity measurement tag | SiteWise measurement property | MQTT event + `MeasurementReading` | `simulation/humidity_excursion.py`, `storage/local_sitewise.py` |
| **Alert limit** | First threshold — notify, monitor more closely | SiteWise alarm / CloudWatch alarm | `alert_limit` float (55 %RH) | `simulation/humidity_excursion.py`, `assurance/state_of_control.py` |
| **Action limit** | Second threshold — action required, GMP impact | SiteWise alarm | `action_limit` float (60 %RH) | same |
| **UNS topic** | Industrial namespace path for a data point | AWS IoT Core MQTT topic | Mosquitto topic | `ot/uns.py` |
| **Sparkplug envelope** | Structured MQTT payload with lineage + quality | AWS IoT Core message schema | `SparkplugInspiredEnvelope` | `ot/sparkplug_envelope.py` |
| **Edge gateway** | OT collector / protocol bridge | AWS IoT Greengrass component | Python poll loop | `ingestion/edge_gateway.py` |
| **OPC UA server** | BMS/EMS/SCADA data server | OPC UA endpoint on-premises | `asyncua` simulator | `ot/opcua_server_sim.py` |
| **Asset model** | Schema for an asset type's measurements | SiteWise AssetModel | `AssetModel` | `storage/local_sitewise.py` |
| **Asset** | Instance of an asset model in the hierarchy | SiteWise Asset | `Asset` | `storage/local_sitewise.py` |
| **Transform** | Computed property from raw measurement | SiteWise Transform | `classify_humidity_state()` | `storage/local_sitewise.py` |
| **Metric** | Aggregated property (duration, average, etc.) | SiteWise Metric | `compute_action_excursion_minutes()` | `storage/local_sitewise.py` |
| **State of control** | Validated condition: are process parameters within limits? | Lambda / Step Functions | State-of-control engine | `assurance/state_of_control.py` |
| **STATE_OF_CONTROL_BREACH** | A GMP-significant excursion above action limit | EventBridge event | `AssuranceEvent(type=STATE_OF_CONTROL_BREACH)` | `models/canonical.py`, `assurance/state_of_control.py` |
| **Canonical event** | Normalized, typed event flowing through the assurance pipeline | DynamoDB / S3 canonical store | `CanonicalEvent` / `AssuranceEvent` | `models/canonical.py` |
| **Assessment** | Structured result of evaluating a stream | Step Functions output | `StateOfControlAssessment` | `models/canonical.py` |
| **Event router** | Routes events to handlers based on type | AWS IoT Rules Engine | `EventRouter` | `ingestion/event_router.py` |
| **Evidence source** | Provenance record for a source data point | S3 / Neptune lineage | `EvidenceSource` | `models/evidence.py` |
| **Evidence item** | Reference to a piece of audit evidence | S3 object / Neptune node | `EvidenceItem` | `models/evidence.py` |
| **Control plane / API** | Persona views for QA, engineering, plant head | API Gateway + Lambda | FastAPI app | `api/main.py` |

## Areos language map

| Areos phrase | What it means in code |
|---|---|
| "Sell proof, not monitoring" | The output is `StateOfControlAssessment` + `EvidenceSource`, not just a raw tag value |
| "Connect signals to validated state" | `classify_humidity_state()` + `run_humidity_state_of_control()` |
| "Utility event → batch → quality risk" | `humidity_excursion.py` links `utility_asset` → `batch_id` → `product_id` |
| "Human-approved evidence" | Assessment is generated by engine; approval step is a future workflow |
| "Read-only-first" | `EvidenceSource.read_only = True`; edge gateway reads but never writes |
| "Source lineage" | `source_lineage` dict on `StateOfControlAssessment` |
| "Tenant/site awareness" | Every model carries `tenant_id` and `site_id` |
