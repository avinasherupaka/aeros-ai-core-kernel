# Bedrock Runtime Architecture

## Core statement

Amazon Bedrock is the **interface and rendering layer**, not the deterministic decision engine.

Regulated conclusions must be generated by deterministic Areos algorithms and supplied to Bedrock as grounded, structured inputs.

## Bedrock responsibilities

Bedrock may be used for:
- narrative rendering of deterministic answers
- evidence package summarization
- question routing to deterministic tools
- retrieval from approved OpenSearch / Bedrock Knowledge Base sources
- deviation or CAPA draft text generation for human review

## Bedrock non-responsibilities

Bedrock must not:
- invent missing evidence
- fabricate batch IDs, timestamps, or laboratory results
- autonomously release product
- autonomously close deviations or CAPAs
- claim the platform is automatically compliant
- issue write/control commands to OT systems

## Runtime flow

1. user asks a question
2. control plane authenticates and authorizes the request
3. deterministic tool registry selects the right governed function
4. deterministic algorithm returns a structured answer object
5. Bedrock receives only the approved answer object and approved citations
6. Bedrock renders an explanation or draft within grounding constraints
7. local/application guardrails validate output before it is shown
8. human reviewer decides on any GxP action

## Grounding policy

Bedrock output must be constrained to:
- cite only provided evidence
- report missing evidence explicitly
- preserve deterministic answer identifiers
- require human review for all GxP decisions
- avoid unsupported compliance claims
- avoid OT write/control instructions

## Knowledge retrieval stance

OpenSearch and Bedrock Knowledge Bases are useful for semantic retrieval of approved SOPs, validation references, and contextual documents. They remain a **retrieval convenience layer**, not regulated truth.

## Runtime boundary

### Local sandbox
- simulation only
- no AWS credentials required
- used to validate contracts, prompt constraints, and guardrail logic

### AWS-native tenant-site cell
- production runtime target
- controlled access to deterministic tools and approved retrieval layers
- governed audit and security controls

## Compliance language

The Bedrock runtime is **designed to support 21 CFR Part 11 / GxP controls, validation evidence, auditability, electronic-record integrity, and customer CSV** by keeping regulated conclusions deterministic and human-approved. It does not itself create automatic compliance.
