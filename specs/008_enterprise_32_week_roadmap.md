## Week/day learning and build cadence

### Weekly rhythm
- **Mon:** read architecture + ontology/assurance docs
- **Tue:** implement scoped code/IaC changes
- **Wed:** run targeted tests and demos
- **Thu:** capture validation evidence/runbook updates
- **Fri:** review backlog, risks, and next-week scope

### Example daily learning-to-code schedule for a new team member
- **Day 1:** study `README.md`, `docs/runbooks/00_start_here.md`, `docs/architecture/tenant_site_cell_architecture.md`
- **Day 2:** run local sandbox simulations and Week 1–5 tests
- **Day 3:** inspect connectors, UNS topics, and Greengrass V2 docs
- **Day 4:** inspect ontology and scenario library
- **Day 5:** run assurance engine tests and API demo endpoints
